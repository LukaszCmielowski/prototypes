AutoRAG → OpenRAG: find optimal settings

Use AutoRAG (ai4rag) to search for the best embedding model, language model, chunk size, and retrieval settings for your OpenRAG pipeline (Langflow + Docling + Opensearch). Apply the winning config directly to OpenRAG.

Integration approach

OpenRAG has fixed pipeline steps (Docling parsing, chunking, embedding, Opensearch index, retrieval, LLM). AutoRAG optimizes the configurable parameters of those steps so you don't have to guess — run optimization once, then apply the recommended settings to your OpenRAG instance.

  • Embedding model — AutoRAG tries several; best one for your data becomes the OpenRAG embedding step.
  • Language model — Which LLM to use for generation; AutoRAG scores combinations.
  • Chunk size / strategy — Chunk token size, overlap; maps to OpenRAG chunking config.
  • Retrieval — top_k, similarity threshold; applied to OpenRAG retrieval step.
